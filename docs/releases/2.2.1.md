# OpenVitals 2.2.1

Released 2026-07-13.

A packaging release, and a short one. OpenVitals no longer contains a line of Google proprietary code, which is what it takes for F-Droid to build it.

### Changed

- **Google Play Services are gone.** The location plugin depended on Google's proprietary `play-services-location` library, and the app never used it. Route recording has always asked Android's own location manager for satellite fixes directly, because Google's fused provider quietly mixes in network and wifi positions — and those sail through an accuracy check without being the GPS readings a recorded route is actually made of. So the library sat in every build being nothing but a dependency, and it was the one thing standing between OpenVitals and F-Droid, whose scanner refuses to build an app that carries it. The plugin is now vendored without it. Nothing about recording changes, because the code path this removes was never the one that ran.

Nothing else changed. For anyone installing from Codeberg or Google Play there is no behavioural difference between 2.2.0 and 2.2.1; the release exists so that the app can be built, from source, by a build server that will not touch Google's libraries.

The Codeberg and Play builds keep the same package name and signing certificate, so this installs as a normal update.
