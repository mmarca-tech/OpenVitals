# OpenVitals 2.0.1

Released 2026-07-11.

OpenVitals is now built on Flutter. Same app, same features, same Health Connect data - and the foundation for an iOS version later. (2.0.0 was prepared but never published, so this release carries everything since 1.9.0.)

**Your settings carry over.** A one-time migration on first launch brings across your goals, units, theme, caffeine profile, body and heart-rate profile, custom drinks, paired sensors, reminders, activity notes, home-widget setup, and offline map packs. Health Connect data is stored outside the app and is untouched. Two things reset and have to be set again: the order of your dashboard tiles, and the drink chosen for a quick-beverage widget.

### New

- **Activity splits.** Any activity with a distance is broken into segments - 1 km by default, configurable in Settings - showing pace, average heart rate, elevation, and how each segment compares to the activity's own average. Laps recorded by a watch are shown exactly as recorded. A treadmill run with no per-time data is split evenly and labelled as an estimate rather than pretending to be a measurement.

### Fixed

- Missing heart rate at the start of a recorded activity. Health Connect filters heart-rate records by the record's own boundary, so a record that opened before the workout was dropped whole.
- The dashboard showed the default daily goals instead of yours - a 6,000-step goal still read "of 8,000" and filled the ring against 8,000. It affected all fourteen goals, not just steps.
- The beverage breakdown showed the app's package name instead of the drink's name.
- The Day view listed no beverages at all.
- The metric carousel stopped swiping after you dragged a tile to another page, until the app was restarted.

### Faster

- Apple Health imports skip records in categories you did not select before ever reading them, and progress now advances while the export is scanned instead of sitting at 0%.

### Changed

- **Hydration and mindfulness reminders may arrive a few minutes later than the time you set.** Google restricts exact alarms to alarm-clock and calendar apps, so reminders are now delivered within a short window instead of on the dot.

The app keeps the same package name and signing certificate, so this installs as a normal update.
